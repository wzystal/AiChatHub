# 📄 AI Chat Android Demo 需求文档（流式输出 + UI优化版）

---

# 1. 项目名称

**LightChat - Android AI Streaming Chat Demo**

---

# 2. 项目目标（非常关键）

本项目用于验证：

> ⭐ Android 端如何实现 AI 流式输出（SSE） + 高性能消息列表渲染优化

重点不是功能完整，而是：

- 流式 token 更新机制
    
- UI 局部刷新优化方案
    
- Android 架构设计能力（MVVM）
    

---

# 3. 核心功能范围（MVP）

## 3.1 基础聊天功能

- 支持用户输入问题
    
- 展示聊天消息列表（RecyclerView）
    
- 支持 user / assistant 两种消息类型
    

---

## 3.2 AI 流式回复（核心）

- 调用大模型 API（OpenAI / 兼容接口）
    
- 使用 **SSE（Server-Sent Events）或 chunk 流式返回**
    
- assistant 消息逐 token 更新
    

效果：

```
用户：你好
AI：你 -> 你是 -> 你是一个 -> 你是一个AI助手
```

---

## 3.3 消息列表动态更新（核心优化点）

实现要求：

- assistant 消息实时更新，但**不刷新整个列表**
    
- 仅更新“最后一条消息 item”
    

---

## 3.4 聊天历史（轻量）

- 当前 session 内存保存
    
- 可选：使用 SharedPreferences / Room（简单存储）
    

---

# 4. 非功能需求（重点：面试价值来源）

## 4.1 UI性能优化（核心）

必须实现以下优化策略：

### ⭐（1）局部刷新策略

- 使用 `notifyItemChanged(position, payload)`
    
- 避免 `submitList()` 全量刷新
    

---

### ⭐（2）流式更新控制

- 每次 token 更新：
    
    - 只更新最后一个 item
        
    - 不触发全列表刷新
        

---

### ⭐（3）UI渲染优化策略

- 流式阶段使用 **纯文本渲染**
    
- AI生成完成后再做 Markdown 渲染
    

---

### ⭐（4）更新频率控制（重要）

- 不建议每个 token 都刷新 UI
    
- 使用简单 throttle（如 50~100ms）
    

---

## 4.2 架构要求

采用 MVVM：

```
UI (Activity/Fragment)
↓
ViewModel
↓
Repository
↓
Network (SSE API)
```

---

# 5. 技术方案（建议实现方式）

---

## 5.1 网络层

### 推荐方案（优先）

- OkHttp + SSE（EventSource）
    

或：

- OkHttp streaming response body
    

---

## 5.2 UI层

- RecyclerView + Adapter
    
- DiffUtil（可选）
    
- payload 局部更新
    

---

## 5.3 状态管理

- StateFlow（推荐）
    
- 或 LiveData（备选）
    

---

## 5.4 数据结构设计

```kotlin
data class ChatMessage(
    val id: String,
    val role: Role, // USER / ASSISTANT
    var content: String,
    val isStreaming: Boolean = false
)
```

---

# 6. 核心交互流程（必须实现）

## 6.1 用户发送消息

```
User input → ViewModel → Repository → API
```

---

## 6.2 流式返回流程（重点）

```
API返回token chunk
        ↓
Repository接收流式数据
        ↓
ViewModel更新最后一条 message
        ↓
StateFlow触发 UI 更新
        ↓
RecyclerView notifyItemChanged(lastIndex, payload)
```

---

## 6.3 UI更新逻辑（核心）

### 只允许更新最后一条消息：

```kotlin
messages[lastIndex].content += token
adapter.notifyItemChanged(lastIndex, PAYLOAD)
```

---

# 7. Markdown渲染策略（可选但推荐）

## 规则：

- assistant 消息支持 Markdown
    
- user 消息纯文本
    

---

## 渲染策略：

|状态|渲染方式|
|---|---|
|streaming中|plain text|
|完成后|Markdown|

---

工具建议：

- Markwon（Android Markdown库）
